## 1순위: 아파트 전월세 실거래가 (전세가율) — ✅ 완료 (2026-07-09)

**진행 상황 (2026-07-09)**: `market_agent/collectors/molit_rent.py`, `market_agent/collectors/data_go_kr.py` 작성. 템플릿(`index.html`)에서 실거래가/전세가율 카드를 모두 보여주도록 수정. 테스트 12개 추가, 전체 52개 통과. data.go.kr 전월세 API 활용신청 승인 완료, 실제 키로 원본 응답과 통합 수집기 둘 다 검증 완료 — End Point(`RTMSDataSvcAptRent`)와 필드명(`deposit`, `monthlyRent`, `excluUseAr`)이 코드와 정확히 일치해서 추가 수정 없이 바로 동작함. (자세한 내용은 `PROJECT_NOTES.md`의 2026-07-09 항목 참고)

**왜 필요한가**: 전세가율(전세가/매매가)은 실수요 강도와 갭투자 리스크를 보여주는 핵심 지표. 매매 실거래가는 이미 반영했으니, 같은 API 계열의 전월세 자료만 추가하면 됨.

**데이터 소스**: 국토교통부_아파트 전월세 실거래가 자료 (공공데이터포털)
- https://www.data.go.kr/data/15126474/openapi.do
- 매매 API(RTMSDataSvcAptTrade)와 같은 방식으로 활용신청 (data.go.kr 계정에서 신청, 승인 보통 즉시~몇 시간)
- 파라미터도 매매랑 동일한 패턴일 가능성 높음: `LAWD_CD`(법정동코드 5자리), `DEAL_YMD`(계약년월 6자리)
- ⚠️ 주의: 매매 API 때 End Point 이름(`RTMSDataSvcAptTradeDev` → 실제로는 `RTMSDataSvcAptTrade`)과 성공 코드(`00`이 아니라 `000`)가 예상과 달라서 버그가 났었음. 전월세 API도 활용신청 상세 페이지의 "End Point"와 "일반 인증키" 화면을 스크린샷으로 먼저 확인하고 시작할 것.

**구현 방법**: `market_agent/collectors/molit.py`와 거의 같은 구조로 새 모듈(`rent.py` 또는 `molit.py`에 클래스 추가) 작성.
- 응답 필드는 매매(`dealAmount`, `excluUseAr`)와 다르게 보증금(`deposit` 또는 `보증금액`), 월세(`monthlyRent`), 계약구분(전세/월세) 필드가 있을 것 (정확한 필드명은 활용신청 후 Swagger 문서에서 확인)
- 전세가율 계산: 같은 동/기간의 전세 중위가 ÷ 매매 중위가 × 100
- 새 `EvidenceItem` category 예: `"rent_data"` 또는 기존 `"market_data"`에 통합하고 tags로 구분
- `rule_engine.py`에 새 카테고리 가중치/캡 추가 필요 (market_data 추가했던 패턴 그대로 참고)
- 테스트: `test_molit.py` 패턴 그대로 (FakeClient, XML 파싱, 전세가율 계산 회귀 테스트)

**예상 난이도**: 낮음 (기존 코드 재사용 가능)

## 2순위: 공식 미분양 통계로 키워드 추측 대체 — ⏸️ 보류 (2026-07-09)

**왜 필요한가**: 어제 겪은 버그(미분양 없는 단지가 "미분양"으로 잘못 감점)가 정확히 이 문제. 뉴스 키워드 대신 정부 공식 통계를 쓰면 이 종류의 오탐이 구조적으로 사라짐.

**조사 결과 (2026-07-09), 보류 사유**: 애초에 적어뒀던 "한국부동산원_부동산통계 조회 서비스(15134761)"는 실제로는 미분양을 다루지 않음 — 2022년 API 개편 공지를 확인해보니 이 서비스군은 주간아파트동향/전국주택가격동향조사/실거래가격지수/매입자 연령대별 거래/월세가격동향 등 8종뿐이고 미분양은 포함되지 않았음. data.go.kr에서 "미분양"으로 직접 검색해도 국토부 차원의 전국 통일 API는 없고, 시/도별로 파편화되어 있음:
- 오픈 API 보유: 경기도(`15057206`), 부산광역시(`15040489`), 경상남도(`15062539`) 등 일부 지자체뿐
- 서울은 API 자체가 없음 (전국에서 가장 수요가 많을 지역인데도)
- 국토부 통계누리(stat.molit.go.kr)의 전국 통합 "미분양주택현황보고"도 REST API가 아니라 다운로드(HWP/Excel) 형태

전국 주소 기반 서비스인 이 에이전트 특성상, 일부 시/도만 커버하는 파편화된 API 여러 개를 개별 연동하는 건 유지보수 비용 대비 효용이 낮다고 판단해 사용자와 상의 후 보류. 나중에 재검토한다면 "경기/부산/경남만 부분 지원 + 나머지는 기존 뉴스 키워드 유지" 절충안이 현실적인 시작점.

**예상 난이도**: 중간~높음 (전국 통일 API 없음, 시/도별 개별 연동 필요)

## 3순위: 규제지역(조정대상지역/투기과열지구) 지정 현황 — ✅ 완료 (2026-07-09)

**왜 필요한가**: 지금은 뉴스에 "규제"라는 단어가 있는지로만 판단하는데, 실제로는 국토부가 공식적으로 지정/해제 고시를 냅니다. 확정 정보로 바꾸면 신뢰도가 크게 올라감.

**진행 상황 (2026-07-09)**: 예상대로 공식 오픈 API가 없어서, 계획했던 대로 수동 유지보수 정적 목록 방식으로 구현. `market_agent/regulation_areas.py`에 조정대상지역/투기과열지구 목록 작성 (WebSearch로 확인한 2026-07-09 기준 현황: 서울 25개 자치구 전역 + 경기 15곳). `market_agent/collectors/regulation.py`의 `RegulationAreaCollector`가 좌표의 시/군/구를 이 목록과 대조해 `category="policy"` evidence 생성 — 기존 rule_engine 정책 카테고리 로직을 그대로 재사용해 추가 변경 불필요했음. 외부 API 호출이 없어 별도 키/설정 없이 항상 실행됨. 테스트 8개 추가. **주의**: 이 목록은 사람이 뉴스를 보고 갱신하는 자료라 시간이 지나면 오래될 수 있음 — 국토부가 새로 지정/해제 발표를 할 때마다 `market_agent/regulation_areas.py`를 갱신해야 함 (자동 갱신 아님).

**예상 난이도**: 낮음(수동 리스트 방식) — 다만 "실시간 자동 반영"은 아니고 주기적으로 사람이 갱신해야 함.

## 4순위: 한국은행 기준금리 (거시 맥락) — ⚙️ 코드 완료, 실제 키 검증 대기중 (2026-07-09)

**진행 상황 (2026-07-09)**: 원래 계획했던 "통계코드검색으로 통계표코드+항목코드 확인" 대신, ECOS의 "100대 통계지표(KeyStatisticList)" 서비스를 사용하도록 구현 — 사전 정의된 100개 핵심 지표를 이름("한국은행 기준금리")으로 바로 찾을 수 있어서 통계표코드를 몰라도 됨 (매매 실거래가 API 때 코드/필드 추측이 틀렸던 것과 같은 리스크를 원천적으로 피함). `market_agent/collectors/ecos.py` 작성, sample 엔드포인트(인증키 불필요)로 응답 스키마와 에러 형식은 확인 완료. `AnalysisReport.base_rate_note` 필드로 리포트에 항상 노출(점수 미반영, impact=0). 테스트 9개 추가.

**아직 안 끝난 것**: ecos.bok.or.kr에서 실제 회원가입 + 인증키 발급 후 로컬에서 `EcosClient.fetch_key_statistics()` 실제 호출 테스트 필요 (sample 응답과 실제 응답이 다를 가능성은 낮지만 매매/전월세 API 때 두 번 다 실제로 검증하고 나서야 안심할 수 있었으므로 동일한 절차 권장).

**왜 필요한가**: 개별 단지 데이터는 아니지만, 금리는 부동산 가격에 영향을 주는 가장 큰 거시 변수 중 하나. 리포트에 "지금 금리 환경" 한 줄만 추가해도 맥락이 풍부해짐.

**데이터 소스**: 한국은행 ECOS Open API
- https://ecos.bok.or.kr/api/
- data.go.kr와 별개로 한국은행에서 직접 무료 가입 + 인증키 발급 (보통 1일 이내 승인)

**예상 난이도**: 낮음

## 5순위: 청약 경쟁률 (신규 분양단지 한정)

**왜 필요한가**: 신규 분양 단지라면 실제 청약 경쟁률 숫자가 뉴스 키워드보다 훨씬 직접적인 수요 지표.

**데이터 소스**: 한국부동산원_청약홈 청약접수 경쟁률 및 특별공급 신청현황 조회 서비스
- https://www.data.go.kr/data/15098905/openapi.do

**구현 방법**: 단지명으로 청약 결과를 조회해서 경쟁률 숫자를 evidence로 반영. 이미 분양이 끝나고 입주 중/입주 완료된 단지에는 해당 없음 (신규 분양 단지에만 의미 있음).

**예상 난이도**: 중간
